## `yn-button` 已交付能力（2026-05）

- 语义色：`variant="primary|success|warning|danger|neutral|dark|default"`
- 尺寸：`size="mini|small|medium"`（默认 `medium`）
- 事件：使用原生 `click`
- 文案：使用默认插槽（`<slot></slot>`，支持自定义类名）
- 图标插槽：`prefix-icon` / `suffix-icon`
- Loading：
  - `loading`、`loading-type="left|center|right"`
  - `loading-type="center"` 时保留文本且 loading 层整体居中
  - 支持 `slot="loading"` 自定义 loading（动画由使用方自行控制）
- 命中热区：`hit-slop`（默认开启，`::before` 外扩 5px）
- 禁用态：`disabled`（按 `variant` 匹配禁用配色）
- 背景层：使用内部绝对定位 `span.bg`
- 圆角：通过 `--yn-button-radius` 自定义
- 常用 CSS 变量：`--yn-button-bg`、`--yn-button-hover-bg`、`--yn-button-disabled-bg`、`--yn-button-disabled-color`、`--yn-button-radius`、`--yn-button-loading-size`



选择组件有2个组件搭配使用
父组件：yn-group-pick

value属性：可以接收数组和number或者string，进行回显

change事件：(ids: 选中的id集合， flag: boolean 是否选中)=>{} 因为有的时候是多选这里回传类型直接就定义成函数

multiple: 默认false只能选中一个yn-pick, true的时候可以选择多个yn-pick

selected-icon: 选中状态图标。`selected=true` 时展示。

unselected-icon:未选中状态图标，默认是描边圆环图标，不传会使用默认值。

show-unselected-icon:未选中状态是否展示 unselected-icon，默认 false。


子组件：yn-pick

子组件 yn-pick，点击就触发yn-group-pick的change事件

然后子组件默认插槽会塞入到这个相对div里面，由用户自定义内容

就是一个相对定位 div，命名为 wrap，包裹右上角图标：选中时显示 selected-icon；未选中时仅在 show-unselected-icon=true 时显示 unselected-icon。

border属性：默认值true, wrap 会有一after伪类，并且pointer-events: none，他会设置1px，#000边框，8px圆角,覆盖在默认插槽的元素上

自定义变量控制  border-width border-color 还有圆角
